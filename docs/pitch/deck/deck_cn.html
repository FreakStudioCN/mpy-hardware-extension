02 / 14时代变化

AI 已经开始接管数字世界，但物理世界还停在手工拼装阶段。

数字世界已发生的变化
写代码从补全，到 agent 自己改多文件、跑任务、修错误
做网页 / app从手写页面，到一句话生成原型和完整页面
做图像 / 视频从工具操作，到自然语言生成资产
做硬件仍然要选型、接线、找库、烧录、看串口

AI 真正进入物理世界，必须能把想法变成可运行的设备。

03 / 14用户痛点

创作者不是没有想法，是卡在硬件落地链路。

01

选型难

ESP32 / Arduino / STM32、传感器、执行器、通信方式，选择太多。

02

接线难

I2C / SPI / UART / ADC、电压、引脚复用，任何一步错都跑不起来。

03

驱动难

资料分散在 GitHub、厂商 PDF、论坛和示例代码里。

04

调试难

串口日志、供电、接线、固件版本混在一起，新手很快放弃。

门槛不在“想象力”，而在“把东西真的跑起来”。

04 / 14为什么难

普通软件 agent 到了硬件现场会失效，因为它缺三样东西。

缺口具体表现后果
真实驱动知识MCU、传感器、库版本、固件能力高度碎片化LLM 容易幻觉 API 和安装方式
本地执行权限烧录、串口、文件上传必须访问用户电脑和开发板纯网页 demo 很难形成闭环
硬件状态感知模块不会主动告诉系统“我是谁、能做什么、接在哪”AI 只能猜引脚、猜协议、猜错误原因

核心不是“让 AI 写一段代码”，而是给 AI 一套硬件运行系统。

06 / 14真实数据流

“一句话造硬件”不是聊天框，而是一条工程流水线。

01

用户说需求：温湿度监测，超过阈值亮灯并显示数值。

02

获得硬件画像：绑定设备、I2C 扫描、勾选模块或自然语言描述。

03

能力匹配：显示、传感、通信、执行器等 hardware_tags。

04

关联 Package Intelligence：读取 package.json、README、driver context、依赖和芯片/固件要求。

05

生成 MicroPython 项目：代码、驱动、资源、配置一起生成。

06

打包 .mpk，写入 MANIFEST.JSON，可下载、推送或发布。

07

mpremote 本地运行，读取串口输出并自动修复。

一句话只是入口，壁垒在硬件画像、驱动检索、应用打包、本地运行和反馈修复。

08 / 14Package Intelligence / IDE

Package Intelligence 解决“找包难、找包乱”，IDE 插件把包管理放进真实工作流。

  • 200+ 个 MicroPython 包/驱动，覆盖传感器驱动、通信模块、人机交互等场景。
  • 包详情包含版本、作者、license、芯片支持、固件依赖、文件列表等机器可读信息。
  • Thonny 内搜索、查看元数据、下载缓存，并通过 mpremote 安装到开发板 /lib。
  • 处理多文件包、依赖、网络失败、mpremote 未配置、开发板未连接等错误场景。
200+

真实 MicroPython 包，不是模型幻觉出来的驱动名。

09 / 14运行闭环

MicroPython + mpremote 更适合 AI 反复试错。

闭环环节传统 Arduino / C 路径我们的 MicroPython 路径
写代码.ino 隐式规则、多文件拼接、C/C++ 依赖复杂Python 结构清晰，AI 更容易生成和修复
安装依赖手动找库、复制代码、处理版本冲突包搜索 + IDE 内安装 + 依赖自动处理
运行反馈编译、烧录、串口监视器分散，改一次等一次mpremote run / exec / fs，上传、运行、读日志在一条链路
自动修复用户自己看串口和资料agent 读取错误输出后改代码、重跑、再验证

硬件 AI 的关键差异不是“生成第一版代码”，而是把设备反馈纳入 agent 循环。

10 / 14uPyOS / uPyStore

把一次性代码，变成 MCU 上可安装、可推荐、可分发的应用。

组件具体设计价值
uPyOS 0.9.0MicroPython + LVGL，运行于 ESP32 / ESP32-S3 等 MCU桌面启动器、应用安装卸载、系统设置、WiFi 管理
D-Shell搭载 uPyOS 的硬件终端设备类比 Android 手机，是 uPyOS 的主要运行载体
.mpkuPyOS 应用安装包，本质是固定结构压缩包对标 APK，让硬件应用可以安装、更新、分发
MANIFEST.JSON声明应用名称、版本、入口、发布者等对标 Android 清单文件，保证应用可被系统识别
app_index.json设备端商城静态入口，字段结构必须稳定出厂设备依赖它拉取应用索引，不能破坏兼容

V2 继续接入 device_fingerprint、device_profiles、hardware_tags、modules 注册表，实现硬件画像、能力匹配、推荐、语义搜索和 AI 生成 .mpk。
