# Blockless 新手测试指南

> 一句话:Blockless 是 VS Code 里的一个 AI 插件 —— 你用中文说一句"我想做什么",它帮你选器件、画接线、生成 MicroPython 代码,确认后**直接刷进你手上的开发板**。
>
> 这份文档带你从零跑通一次,然后帮我们试几个例子、把问题反馈回来。
>
> **预计 20–30 分钟。** 你需要懂一点嵌入式(知道什么是 GPIO / I²C / 串口 / 刷固件就够了),**不需要**会写 Python,也**不需要**读源码。
>
> 全程卡住的地方,大多不在硬件,而在 **GitHub 账户 + 网络**这两关 —— 这俩我们讲得最细。

---

## 0. 你要帮我们测的是什么

就一件事:**"打一句话 → 看它生成 → 刷到真板上 → 是否真的能用"** 这条完整链路顺不顺、哪里别扭、哪里看不懂。你的卡点就是我们要修的东西,所以**越卡越有价值**,别客气。

## 1. 开始前,先备齐这些

## 2. 装 VS Code

VS Code 是微软的免费代码编辑器,Blockless 就是装在它里面的插件。

1. 打开 https://code.visualstudio.com/ ,点大大的下载按钮,按你的系统(Windows / Mac)下载。
2. 双击安装,一路"下一步"即可。
3. 装完打开它,看到一个深色的编辑器界面就对了。

## 3. 装 Blockless 插件

1. 在 VS Code 左边那一竖排图标里,点**四个方块**那个(扩展 / Extensions),或按 `Ctrl+Shift+X`(Mac 是 `Cmd+Shift+X`)。
2. 顶部搜索框里**精确输入:`Blockless`**。
3. 在结果里认准**发布者是 `blockless`** 的那一个,点 **Install / 安装**。
4. 装好后,左边工具栏会多出一个 Blockless 图标,点它打开插件面板。

## 4. 注册一个 GitHub 账户

GitHub 是全球程序员放代码的网站。我们**只是借它来登录**(确认"你是你"),不需要你会用它,也不会动你电脑里的东西。

如果你**已经有** GitHub 账户,跳到 [第 5 节](#5-在-vs-code-里登录)。

没有就现注册:

1. 打开 https://github.com/signup
2. 按提示一步步填:
   - **Email**:填你常用的邮箱。
   - **Password**:设一个密码(它会要求够长、够复杂)。
   - **Username**:起一个用户名(英文/数字,别人可能看得到,随意起一个即可)。
3. 它可能让你做一个**小拼图验证**(证明你不是机器人),按提示点几下。
4. GitHub 会往你邮箱发一个**验证码**,回邮箱把验证码填回去。
   - 没收到?等 1–2 分钟,翻一下"垃圾邮件 / 推广"文件夹。
5. 看到进入 GitHub 主页,就注册成功了。

〔截图:GitHub 注册页 + 邮箱验证码那一步〕

> 打不开 github.com 注册页?同样是网络问题,看 [第 6 节](#6-打不开网页怎么办)。

## 5. 在 VS Code 里登录

注册好账户后,回到 VS Code:

1. 打开 Blockless 面板,随便打一句话开始用(比如下面第 9 节的例子)。
2. 第一次使用时,VS Code **右下角会弹一个提示**,大意是"插件想用 GitHub 登录",点 **允许 / Allow / Continue**。
3. 它会**自动打开浏览器**,跳到 GitHub 的授权页,点绿色的 **Authorize / 授权**按钮。
4. 浏览器提示"可以回到 VS Code 了",回到 VS Code 即可 —— **登录就完成了**,不用再手动输账号密码。

## 6. 打不开网页怎么办

**这是最关键、也最容易卡住的一节**,请认真看。

Blockless 要正常工作,你的网络必须能稳定访问两类网站:

- **github.com** —— 注册和登录用;
- **我们的云端服务** —— 插件生成代码时,在后台和它通信(你不用管地址)。

国内网络访问这两类站点**有时会很慢或打不开**。怎么判断是不是网络问题:

- ✅ **像是网络问题**:github.com 打不开/转圈很久;扩展市场搜 Blockless 半天没结果;插件里一直转圈最后报 "Cannot reach the auth API" / 连接失败之类。
- ❌ **不像网络问题**:网页都打得开、就是某一步操作报别的错 —— 那看 [第 11 节 FAQ](#11-常见卡点速查)。

**怎么解决:** 你需要一个能稳定访问国际网站的网络环境。

> *(本文不提供具体的网络配置步骤;请按你们的实际情况补充。)*

确认网页能正常打开后,再回去继续上面的步骤。

## 7. 准备你的开发板

> 这一节默认你懂点嵌入式,所以讲得简略。卡住的话第 8 节的体检会帮你定位。

1. **确认板子型号**是这三块之一:ESP32-S3-DevKitC-1 / ESP32-C3-DevKitM-1 / Raspberry Pi Pico W。
2. **刷好 MicroPython 固件**(只需一次):
   - 去 https://micropython.org/download ,搜你的板子型号,下载对应固件,按它的说明刷进去(ESP32 系列用 esptool;Pico W 直接拖 `.uf2`)。
   - 第 8 节的体检如果发现板上没有 MicroPython,也会**直接给你该板的固件下载链接**。
3. **用数据线把板子插到电脑** USB 口。
   - ESP32 板子通常还需要 **USB 转串口驱动**(CH340 或 CP2102,看你板子上的芯片),没识别到串口就先装驱动。
4. **装好 Python 3.10+**:插件靠本机 Python 和板子通信。没装的话去 https://www.python.org/downloads/ 下载安装(Windows 安装时记得勾上 "Add Python to PATH")。

## 8. 用"环境体检"自查(强烈建议)

记不住上面那一堆?Blockless 面板里有一个 **环境(Doctor)** 标签页,点一下就**逐项体检**,这是你最好的自查工具:

| 它检查的 | 绿了代表 | 没绿它会怎么做 |
|---|---|---|
| **Python** | 本机装好了 Python | 给你 python.org 下载链接 |
| **依赖** | 通信用的依赖就绪 | 给一个"**安装依赖**"按钮,点一下自动装 |
| **板子** | 检测到一个串口设备 | 提示"没检测到/检测到多个,请选一个" |
| **MicroPython** | 板上有 MicroPython 固件 | 给你**该板的固件下载链接** |

> 用法:把板子插好,打开环境标签,点 **重新检查 / Re-check**。检查 MicroPython 那一步会短暂打断板上正在跑的程序,属正常。
>
> 这一页本身也是要测的重点之一:**哪一项的提示你看不懂、或它说绿了但其实没装好,都记下来反馈给我们。**

## 9. 试一试:照着抄这几句

四项体检都 ✅ 后,就可以真正生成 + 刷板子了。建议**从第 1 个开始**(不需要额外元件,先把"能刷进板子"这条链路跑通),再按手头的元件试后两个。

在 Blockless 的输入框里,**直接粘贴下面任意一句中文**:

1. **最简单(只要一块板,不用接任何元件):**
   > `让板载 LED 每隔一秒闪烁一次`

2. **接一块 I²C OLED 屏(SSD1306):**
   > `用一块 SSD1306 OLED 屏显示 "Hello Blockless"`

3. **接一个温湿度传感器(DHT11):**
   > `每隔两秒读取一次 DHT11 的温度和湿度,并打印出来`

**正常情况下,你应该依次看到插件做这些事**(看不到/卡住就记下来反馈):

1. 它**选好器件**,告诉你用到哪些零件;
2. 生成一个**项目清单(manifest)**;
3. **推导接线**(哪个脚接哪个脚);
4. **生成 MicroPython 代码**;
5. 问你**要不要部署到板子** —— 你确认后,它把代码刷进去;
6. 看板子上的实际效果:LED 闪了 / 屏上出字了 / 串口里打印出温湿度。

> 元件接线请按它给的接线图来。接好再确认部署。

## 11. 常见卡点速查(FAQ)

**Q:搜不到 Blockless 插件 / github.com 打不开 / 一直转圈最后报连接失败。**
基本都是网络问题,回到 [第 6 节](#6-打不开网页怎么办)。

**Q:登录弹窗一闪而过,或者根本没看到。**
重新在插件里发一句话触发登录;或在 VS Code 左下角的**账户**图标里手动登录 GitHub。还是不行就重启一次 VS Code。

**Q:板子插上了,但体检说"没检测到设备"。**
① 换一根**数据线**(很多线只能充电);② ESP32 板子装好 **CH340/CP2102 驱动**;③ 换个 USB 口再点"重新检查"。

**Q:体检说板上没有 MicroPython。**
按它给的链接下载该板固件刷一次(见 [第 7 节](#7-准备你的开发板)),刷完再"重新检查"。

**Q:体检说"检测到多个设备"。**
说明电脑上插了不止一个串口设备,按提示选你板子对应的那个端口。

**Q:提示额度 / credits 用完了。**
把这条连同你的 GitHub 用户名反馈给我们(见 [第 10 节](#10-遇到问题这样反馈给我们)),我们看一下。

**Q:Python 装了,但体检还是说找不到。**
Windows 上重装 Python 时勾上 **"Add Python to PATH"**,然后**完全退出并重开 VS Code**(只是刷新窗口不够)。
