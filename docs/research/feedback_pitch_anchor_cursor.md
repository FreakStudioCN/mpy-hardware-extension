Pitch deck 的故事脊柱不要 anchor 到 KS 爆款销量（UGREEN NASync / eufy S1 Pro / Carvera Air）。要 anchor 到 Cursor / Copilot / Replit 这一档"AI 重新定义创作工具"的品类叙事。

**Why：** Cursor 估值不是因为它是更好的 VS Code，是因为它"AI-native 重新定义了 IDE"。我们对标的是这个 narrative level，不是对标卖了 $5M 的 KS 硬件。用户 2026-04-29 原话："我们对标的不是 cursor 这种级别叙事吗 对标其他产品不太对吧？"

**How to apply：**
- pitch deck 的 "Why this wins" / 竞品 slide 把 Cursor / Copilot / Replit 当**姊妹品类**（"他们做软件，我们做物理"），不当竞品
- 老 dev kit（Wio Terminal / M5Stack / Arduino / Adafruit / Seeed）只在 problem slide / status quo 章节出现一次，标注为"AI 之前的工具"
- TAM / why now slide 用 Cursor / Copilot 估值/付费数据做"AI 工具吃市场"的论据，而不是 KS 硬件销量
- KS 是 GTM 战术（roadmap 章节用），不是公司身份，不进核心叙事
- Plaud / Humane / Limitless 单 SKU 失败案例只放在 long-tail 章节作"单产品没护城河"反例，不当对标
