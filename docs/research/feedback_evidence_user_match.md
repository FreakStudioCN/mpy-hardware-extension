任何"X 公司销量好 / 融资多 / ARR 高"想当我们 demand 论据时，**先问一句：X 卖给谁？我们卖给谁？是同一批人吗？**

**Why：** 用户 2026-04-29 抓到 deck v2 的逻辑漏洞 —— 用 Plaud / Humane / Limitless / Friend 这些 B2C 成品 AI 硬件销量当我们的 demand 验证。Plaud 卖给"想买录音笔的消费者"，我们卖给"想自己做硬件的创作者"，**两个市场不交叉**。用户原话："plaud 和大众有什么关系吗？就好比 anthropic 很强 和 vibecoding 有什么关系吗？逻辑缕一缕"

**How to apply：**
- 引 demand 数据前做"用户群体对照"：他们的客户是什么人？我们的客户是什么人？是不是同一批？
- AI 硬件 B2C 成品销量数据（Plaud / Humane / Friend / Limitless / Rabbit）**只在我们也定位为"卖给消费者的成品"时**才能用
- 我们做"创作者工具/平台"时应引用同类用户行为的论据 → **Cursor / Copilot / Replit / Squarespace / Wix / Roblox / Shopify / YouTube creator economy** 这类"让 N 个普通人能创造 X"的成功
- 单 SKU vs 平台的论点也要建立在**同一用户群体**上（YouTube 比 broadcasters 这两边的观众是同一批，所以类比成立；Plaud 用户 vs 我们的 maker 用户不是同一批，所以类比不成立）
- 反例红线：不要用"AI 硬件总融资 $1B+"当 demand 论据 —— 那个 demand 是消费者要 AI 硬件成品，不是创作者要硬件创作工具
